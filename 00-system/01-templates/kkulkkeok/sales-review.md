# 격주 판매 리뷰 분석 템플릿 — 꿀꺽하우스

> 격주 제조팀·펍팀 회의에 들고 들어가는 판매 리뷰 보고서를 만들기 위한 지시서.
> 한 줄 호출(`@templates/kkulkkeok/sales-review.md 관점으로`)만으로, **원인·의사결정 후보·액션이 담긴 보고서**가 나와야 한다.
> 표와 숫자만 나열하면 미완성. 숫자 옆 "그래서 무슨 의미" 한 줄이 없는 줄은 다시 채운다.

---

## 사용 데이터 (반드시 교차)

| 파일 | 쓰는 이유 |
|------|----------|
| `sales.csv` | 채널별·일자별·SKU별 매출의 기반 |
| `product_master.csv` | 원가·채널별 단가·수수료율 — **매출만 보지 말고 "남는 장사였나"까지 교차** |
| `content.csv` | SNS·펍 이벤트 게시일 — **매출 급등일과 겹치는지 연결** |
| `inventory.csv` | 현재재고·안전재고·소진일수 — **잘 팔리는데 재고 바닥인 SKU 경보** |

매출(sales) × 마스터(원가·수수료) × 콘텐츠(연동일) × 재고(소진)를 한 보고서 안에서 교차한다. 단일 파일 분석은 이 템플릿의 목적이 아니다.

---

## 출력 구조

### 1. 헤드라인 KPI (4장)
카드 4개: **기간 총매출 / 순마진(수수료·원가 차감 후) / 채널 1위 / 시그니처 SKU 1종**.
- 각 카드는 수치 + **"그래서 무슨 의미" 한 줄** 필수. 예: "순마진 ₩X (총매출의 N%) — 브루펍 직접판매 비중이 높아 수수료 누수가 적었음".
- 순마진 = Σ(수량 × (채널단가 − 원가)) − Σ(채널단가 × 수량 × 채널수수료율). 마스터에서 채널별 단가·수수료율을 가져와 계산.

### 2. 채널별 판매 추이 + 순마진 (핵심 표)
채널 4종(브루펍포스 · 자사몰 · 스마트스토어 · B2B유통) 행, 컬럼: 매출 / 매출비중 / 평균객단가 / 수수료 / 원가 / **순마진** / 순마진율.
- 표 아래 **원인 3~4줄**: 왜 이 채널이 위로 왔나(주말·펍 영업일·이벤트·시즌, 추정은 "(추정)" 표기). 단순 1문장 금지.
- **수수료/원가 교차 강제**: 매출 1위 채널이 순마진 1위가 아닐 수 있다 — 자사몰·스마트스토어는 수수료가, B2B는 공급가 자체가 마진을 깎는다. "매출은 컸지만 남는 건 적었던" 채널을 반드시 짚는다.

### 3. SKU별 판매 + 시즈널 후보
SKU 6종 표: 판매량 / 매출 / 순마진 / 순마진율 / 시즌 태그.
- **TOP만이 아니라 공통점 + 하위 위험군**: 왜 상위 SKU가 위로 왔는지(시그니처·이벤트 연동·시즌) + 부진 SKU·순마진율 낮은 SKU도 짚는다.
- **시즈널 후보 도출**: 마스터의 시즌 태그(여름·가을겨울 등)와 현재 판매 흐름을 교차해, 다음 격주에 밀 시즌 제품 1~2종을 근거와 함께 제안. 예: "산뜻(여름)이 자사몰에서 상승세 — 다음 시즌 푸시 후보".

### 4. 콘텐츠·이벤트 ↔ 매출 연동 (데모 핵심)
content.csv의 연동SKU가 있는 게시일과 sales의 일 총매출·해당 SKU 판매를 교차.
- 검출 규칙(아래) 통과한 연동일을 표로: 게시일 / 콘텐츠 유형·주제 / 연동SKU / 그날 일매출 배수 / 그날 SKU 판매 배수 / 펍예약.
- 각 연동일에 **해석 2~3줄**: "이 릴스/이벤트가 이 SKU를 이만큼 끌었다 — 다음에도 반복할지". 콘텐츠 유형(릴스·캐러셀·스토리·피드·펍 이벤트)별로 어떤 게 매출을 끌었는지 패턴을 짚는다.
- 저성과·비연동 콘텐츠도 한 줄: "이건 도달은 났지만 매출로 안 이어짐".

### 5. 재고 발주 경보
inventory.csv에서 발주상태(발주필요·발주임박) SKU를 `{component.alert-row}` 형태 표로: SKU / 현재재고 / 안전재고 / 일평균출고 / 소진일수 / 리드타임 / 상태.
- **잘 팔리는데 재고 바닥인 SKU를 최우선**으로 올린다. 소진일수 < 발주리드타임(14일)이면 "지금 발주 안 하면 결품" 경보.
- 양조 리드타임이 길다(발효 포함 14일)는 점을 해석에 넣는다 — 펍에서 잘 나가는 시그니처가 결품되면 그날 매출 직격.

### 6. 회의용 코멘트 후보 (3종 미리 제시)
"코멘트 자리 비워두기" 금지. 격주 회의(제조팀·펍팀)에서 바로 고를 수 있게 3종을 미리 뽑는다:
- **숫자형**: "이번 격주 총매출 ₩X, 순마진율 N%. 브루펍 비중 M%."
- **흐름형**: "산뜻 자사몰 런칭이 통했고, 텃밭은 재고가 위험. 다음 격주는 시즌 제품 푸시 + 텃밭 발주."
- **헤드라인형**: "콘텐츠 4건 중 펍 이벤트 2건이 매출을 끌었다 — 펍팀·제조팀이 같은 캘린더를 봐야 한다."

### 7. 액션 (4열 표)
| 담당 | 액션 | 기한 | 우선순위 |
표 아래 각 액션에 **성공 기준** 한 줄. "3줄 이내"로 뭉개지 말 것.
- 제조팀·펍팀으로 담당을 갈라 적는다(이 회사의 긴장은 두 팀 회의에서 온다). 예: 제조팀 = 텃밭 발효 배치 시작 / 펍팀 = 산뜻 펍 푸시.

## 검출 규칙 (이상 신호 임계값)

- **매출 급등일**: 일 총매출 ≥ 기간 일평균 × 1.30 (+30%↑).
- **SKU 급등**: 그날 해당 SKU 판매량 ≥ 기간 일 중앙값 × 2.0.
- **콘텐츠 연동 확정**: 위 두 조건을 동시에 만족하고 content.csv에 같은 날(0일 선행 포함) 연동 게시물이 있을 때.
- **마진 누수**: 채널 순마진율 < 전체 순마진율 − 10%p (수수료·원가가 과하게 깎은 채널).
- **재고 경보**: 현재재고 ≤ 안전재고(발주필요) / 소진일수 ≤ 리드타임 14일(발주임박).
- **부진 SKU**: 판매량 하위 + 순마진율 하위 동시 → 단종/리뉴얼 검토 후보(추정 표기).

## 길이 제약

A4 3~4장. 단, **해석·원인 줄은 생략 금지** — 표를 줄이더라도 "그래서 무슨 의미"는 남긴다. 숫자만 빽빽한 4장보다 표 + 해석이 균형 잡힌 3장이 낫다.

## 톤 가이드

- 보고 대상: 격주 **제조팀·펍팀 합동 회의**. 두 팀이 같은 데이터를 보고 다음 격주 할 일을 가른다.
- 숫자 옆 "그래서 무슨 의미"를 항상 붙인다. 임원 보고서가 아니라 두 팀이 함께 결정하는 자료다.
- 추정은 "(추정)"으로 표기. 데이터로 단정할 수 있는 것과 해석을 구분한다.
- PDF로 뽑을 때는 `@templates/kkulkkeok/design.md` 디자인(비취색·물결 곡선·아치) 적용.
